# Spring AI 改造方案

> 版本：v1.0  
> 日期：2026-05-23  
> 作者：Boya

---

## 一、背景与目标

### 1.1 现状

后端 `api` 项目当前 AI 功能由自研 `com.yxboot.llm` 包（约 68 个文件）承载，其设计思想和接口命名刻意对标 Spring AI，但实际上是一套完全独立的 HTTP 封装实现：

- **Chat 层**：手写 `SseClient` + OkHttp 调用智谱 REST API，通过 `Flux` 实现流式输出
- **Embedding 层**：手写 `HttpClient` 调用智谱 Embedding API
- **VectorStore 层**：手写 Qdrant REST API 客户端（非 gRPC）
- **文档处理层**：自实现 PDF/Word 加载器、字符/章节切分器
- **RAG 层**：`ChatController.enhancePromptWithKnowledge` 中手动编排检索-增强流程

**核心问题**：约 68 个文件的自研框架需长期维护，且仅支持智谱 AI 一个提供商；缺少 Spring AI 生态的标准化、可观测性、扩展支持。

### 1.2 目标

用 **Spring AI 1.0.0** 替换 `com.yxboot.llm` 自研框架，保留现有业务逻辑（多租户、对话管理、文档处理流程），获得：

- 标准化的 AI 抽象层（`ChatModel`、`EmbeddingModel`、`VectorStore`）
- 官方 ZhipuAI / OpenAI 等 Provider 实现，零手写 HTTP
- 内置 RAG Advisor（`QuestionAnswerAdvisor`）
- 未来可低成本接入更多模型提供商
- 删除约 68 个自研文件 + OkHttp + Retrofit 依赖

## 二、现状架构与目标架构对比

### 2.1 现状架构

```
业务层 (modules.ai / modules.dataset / modules.app)
    ↓
Client 连接层 (llm.client.*)       ← 缓存、编排
    ↓
通用抽象层 (llm.chat / llm.embedding / llm.vector / llm.document)
    ↓
Provider 层 (llm.provider.zhipu.*)  ← ZhipuAIChatModel / ZhipuAIEmbeddingModel
    ↓
HTTP Kernel (util.HttpClient / util.SseClient + OkHttp)
    ↓
外部服务 (智谱 REST API / Qdrant REST API)
```

### 2.2 目标架构

```
业务层 (modules.ai / modules.dataset / modules.app)  ← 基本不变
    ↓
租户模型注册表 (ai.registry.*)           ← 新增：多租户 Spring AI 实例管理
    ↓
Spring AI 标准层
  ├── ChatModel (ZhipuAiChatModel)       ← spring-ai-zhipuai-spring-boot-starter
  ├── EmbeddingModel (ZhipuAiEmbeddingModel)
  ├── VectorStore (QdrantVectorStore)    ← spring-ai-qdrant-store-spring-boot-starter
  └── DocumentReader / TextSplitter     ← spring-ai-tika-document-reader
    ↓
Spring AI 内部 HTTP (WebClient / gRPC)
    ↓
外部服务 (智谱 REST API / Qdrant gRPC API)
```

## 三、关键设计决策

### 3.1 多租户动态 API Key 管理

**问题**：Spring AI 的 `ChatModel` / `EmbeddingModel` Bean 在 Spring 启动时由 `@ConfigurationProperties` 固定配置 API Key，而项目中每个租户的 API Key 存储在 DB `provider` 表，需运行时动态获取。

**方案**：保留现有 `ConcurrentHashMap` 缓存模式，但创建的是 Spring AI 官方 Model 实例。

```
ChatModelRegistry（新增）
    └── ConcurrentHashMap<cacheKey, ChatModel>
         ├── key = "zhipuai:{apiKeyHash}:{tenantId}"
         └── value = new ZhipuAiChatModel(new ZhipuAiApi(apiKey), options)
```

不使用 Spring Boot 自动配置的 `ZhipuAiChatModel` Bean（它只支持单一 API Key），而是在注册表中按需手动构造 Spring AI Model 实例。

### 3.2 多租户 VectorStore 集合隔离

**现有策略**：每个知识库对应独立 Qdrant 集合，命名规则为 `tenant_{tenantId}_dataset_{datasetId}`。

**保留策略**：Spring AI 的 `QdrantVectorStore` 支持在构建时指定 `collectionName`，创建一个 `VectorStoreRegistry`，按集合名称缓存 `QdrantVectorStore` 实例：

```
VectorStoreRegistry（新增）
    └── ConcurrentHashMap<collectionName, VectorStore>
         ├── key = "tenant_1_dataset_5"
         └── value = QdrantVectorStore.builder()
                       .collectionName("tenant_1_dataset_5")
                       .embeddingModel(embeddingModel)
                       .build()
```

### 3.3 RAG 检索策略

**现有方式**：在 `ChatController.enhancePromptWithKnowledge` 中手动多库检索、拼接 Prompt。

**目标方式**：对于每个应用对话请求，根据激活的 `datasetIds` 动态构建 `QuestionAnswerAdvisor`（每个 DataSet 对应一个 Advisor），或使用聚合 VectorStore 实现一次多库检索。

由于 Spring AI `QuestionAnswerAdvisor` 目前绑定单一 `VectorStore`，**短期方案**：保留手动检索逻辑，但将检索底层从 `VectorRetrieverClient` 替换为 Spring AI `VectorStore.similaritySearch()`。

**长期方案**：待 Spring AI 支持多 VectorStore 检索后，迁移至 Advisor 模式。

### 3.4 文档处理中的章节切分

`ChapterSplitter`（PDF/Word 结构分析）是项目自研逻辑，Spring AI 标准的 `TokenTextSplitter` 仅做字符/Token 级别切分。改造时：

- **字符切分（`PARAGRAPH` 模式）**：替换为 `TokenTextSplitter`
- **章节切分（`CHAPTER` 模式）**：保留自研 `ChapterSplitter` 逻辑，作为自定义 `DocumentTransformer` 实现，与 Spring AI `Document` 对象兼容

```yaml
      embedding:
        enabled: false         # 禁用自动配置的单例 EmbeddingModel Bean

      qdrant:
        host: ${QDRANT_HOST:localhost}
        port: ${QDRANT_PORT:6334}
        api-key: ${QDRANT_API_KEY:}
        uses-tls: false
```

4. 编译验证，确认无冲突

### 第二阶段：Chat 模型层改造

**目标**：用 Spring AI `ZhipuAiChatModel` 替换 `com.yxboot.llm.provider.zhipu.ZhipuAIChatModel`，保持 `ChatService` 接口不变。

**新增类**：

**`com.yxboot.ai.registry.ChatModelRegistry`**

```java
/**
 * 多租户 ChatModel 注册表
 * 负责为每个 Provider（含租户专属 API Key）创建并缓存 Spring AI ChatModel 实例
 */
@Component
@RequiredArgsConstructor
public class ChatModelRegistry {

    private final ChatModelRegistryConfig config;
    private final ConcurrentHashMap<String, ChatModel> modelCache = new ConcurrentHashMap<>();

    /**
     * 获取或创建指定 Provider 的 ChatModel
     * 使用 (providerName:apiKeyHash:tenantId) 作为缓存键确保租户隔离
     */
    public ChatModel getOrCreate(Provider provider) {
        String cacheKey = buildCacheKey(provider);
        return modelCache.computeIfAbsent(cacheKey, k -> createChatModel(provider));
    }

    private ChatModel createChatModel(Provider provider) {
        String providerName = provider.getProviderName().toLowerCase();
        return switch (providerName) {
            case "zhipuai", "zhipu" -> createZhipuAiChatModel(provider);
            // 未来扩展：case "openai" -> createOpenAiChatModel(provider);
            default -> throw new UnsupportedOperationException("不支持的模型提供商：" + providerName);
        };
    }

    private ChatModel createZhipuAiChatModel(Provider provider) {
        ZhipuAiApi api = new ZhipuAiApi(provider.getApiKey());
        ZhipuAiChatOptions options = ZhipuAiChatOptions.builder()
                .model("glm-4")   // 默认模型，调用时可覆盖
                .build();
        return new ZhipuAiChatModel(api, options);
    }

    private String buildCacheKey(Provider provider) {
        return String.format("%s:%d:%d",
                provider.getProviderName().toLowerCase(),
                provider.getApiKey().hashCode(),
                provider.getTenantId());
    }

    public void evict(Provider provider) {
        modelCache.remove(buildCacheKey(provider));
    }
}
```

**改造 `ChatService`**：

将依赖 `com.yxboot.llm.client.chat.ChatClient` 替换为 `ChatModelRegistry`：

```java
// Before: private final ChatClient chatClient;
// After:
private final ChatModelRegistry chatModelRegistry;

public ChatResponseDTO chatCompletion(Provider provider, ChatRequestDTO request) {
    // 获取 Spring AI ChatModel 实例（按租户 API Key 缓存）
    ChatModel chatModel = chatModelRegistry.getOrCreate(provider);

    // 构建 Spring AI Prompt（消息结构与现有一致）
    Prompt prompt = buildSpringAiPrompt(request);

    // 调用（同步）
    org.springframework.ai.chat.model.ChatResponse response = chatModel.call(prompt);
    return buildResponseDTO(response, provider, request.getModelName());
}

public void streamingChatCompletion(Provider provider, ChatRequestDTO request,
        SseEmitter emitter, Long messageId) {
    ChatModel chatModel = chatModelRegistry.getOrCreate(provider);
    Prompt prompt = buildSpringAiPrompt(request);

    // Spring AI 流式调用：返回 Flux<ChatResponse>
    Flux<org.springframework.ai.chat.model.ChatResponse> stream = chatModel.stream(prompt);

    // 流处理逻辑与现有基本一致（替换类型引用）
    stream.doOnNext(chunk -> { ... })
          .doOnComplete(() -> { ... })
          .doOnError(err -> { ... })
          .subscribe();
}

private Prompt buildSpringAiPrompt(ChatRequestDTO request) {
    List<Message> messages = new ArrayList<>();
    if (StringUtils.hasText(request.getSystemPrompt())) {
        messages.add(new SystemMessage(request.getSystemPrompt()));
    }
    // 历史消息转换（DTO 中的 messages 字段）
    request.getMessages().forEach(m -> messages.add(convertMessage(m)));
    messages.add(new UserMessage(request.getPrompt()));

    ZhipuAiChatOptions options = ZhipuAiChatOptions.builder()
            .model(request.getModelName())
            .temperature(request.getTemperature() != null ? request.getTemperature().doubleValue() : null)
            .topP(request.getTopP() != null ? request.getTopP().doubleValue() : null)
            .maxTokens(request.getMaxTokens())
            .build();
    return new Prompt(messages, options);
}
```

**Message DTO 适配**：

`ChatRequestDTO.messages` 目前使用 `com.yxboot.llm.chat.message.Message`，需转换为 `org.springframework.ai.chat.messages.Message`。可新增 `MessageConverter` 工具类完成转换，或直接修改 DTO 使用 Spring AI 类型。

```java
@Component
@RequiredArgsConstructor
public class EmbeddingModelRegistry {

    private final ConcurrentHashMap<String, EmbeddingModel> modelCache = new ConcurrentHashMap<>();

    public EmbeddingModel getOrCreate(Provider provider, Model model) {
        String cacheKey = buildCacheKey(provider, model);
        return modelCache.computeIfAbsent(cacheKey, k -> createEmbeddingModel(provider, model));
    }

    private EmbeddingModel createEmbeddingModel(Provider provider, Model model) {
        String providerName = provider.getProviderName().toLowerCase();
        return switch (providerName) {
            case "zhipuai", "zhipu" -> {
                ZhipuAiApi api = new ZhipuAiApi(provider.getApiKey());

                        .model(model.getModelName())  // e.g. "embedding-3"

            }
            default -> throw new UnsupportedOperationException("不支持的 Embedding 提供商：" + providerName);
        };
    }

    private String buildCacheKey(Provider provider, Model model) {
        return String.format("%s:%s:%d",
                provider.getProviderName().toLowerCase(),
                model.getModelName(),
                provider.getTenantId());
    }
}
```

### 第四阶段：VectorStore 层改造

**目标**：用 Spring AI `QdrantVectorStore` 替换自研 `com.yxboot.llm.vector.qdrant.QdrantVectorStore`，保持集合命名规则 `tenant_{tenantId}_dataset_{datasetId}` 不变。

**核心变化**：

Spring AI 的 `QdrantVectorStore` 使用 **gRPC**（默认端口 6334）与 Qdrant 通信，而现有自研实现使用 **REST API**（端口 6333）。需确认 Qdrant 实例开启了 gRPC 端口。

**新增类**：

**`com.yxboot.ai.registry.VectorStoreRegistry`**

```java
@Component
@RequiredArgsConstructor
public class VectorStoreRegistry {

    private final QdrantClient qdrantClient;           // Spring AI 自动配置的 gRPC 客户端
    private final EmbeddingModelRegistry embeddingModelRegistry;
    private final ConcurrentHashMap<String, VectorStore> storeCache = new ConcurrentHashMap<>();

    /**
     * 获取或创建指定知识库的 VectorStore
     * 集合名称规则：tenant_{tenantId}_dataset_{datasetId}
     */
    public VectorStore getOrCreate(Long datasetId, Long tenantId, Provider provider, Model model) {
        String collectionName = buildCollectionName(datasetId, tenantId);
        return storeCache.computeIfAbsent(collectionName, k ->
                createVectorStore(collectionName, provider, model));
    }

    private VectorStore createVectorStore(String collectionName, Provider provider, Model model) {
        EmbeddingModel embeddingModel = embeddingModelRegistry.getOrCreate(provider, model);
        return QdrantVectorStore.builder(qdrantClient, embeddingModel)
                .collectionName(collectionName)
                .initializeSchema(true)   // 自动创建集合（替代 autoCreateCollection）
                .build();
    }

    /**
     * 仅用于删除操作（不需要特定 EmbeddingModel）的 VectorStore
     */
    public VectorStore getForDeletion(Long datasetId, Long tenantId) {
        String collectionName = buildCollectionName(datasetId, tenantId);
        // 使用注册表中已有实例，或创建临时实例
        return storeCache.get(collectionName);
    }

    public String buildCollectionName(Long datasetId, Long tenantId) {
        return String.format("tenant_%d_dataset_%d", tenantId, datasetId);
    }

    public void evict(Long datasetId, Long tenantId) {
        storeCache.remove(buildCollectionName(datasetId, tenantId));
    }
}
```

### 第五阶段：文档处理层改造

**目标**：用 Spring AI `TikaDocumentReader` / `PagePdfDocumentReader` + `TokenTextSplitter` 替换 `DocumentProcessorClient` 的文档加载和字符切分逻辑。

#### 5.1 文档加载

**新增工具类** `com.yxboot.ai.document.SpringAIDocumentLoader`：

```java
@Component
public class SpringAIDocumentLoader {

    /**
     * 使用 TikaDocumentReader 加载文档（支持 PDF/Word/TXT 等）
     */
    public List<org.springframework.ai.document.Document> load(Resource resource) {
        TikaDocumentReader reader = new TikaDocumentReader(resource);
        return reader.get();
    }

    /**
     * 使用 PagePdfDocumentReader 按页加载 PDF（保留页码元数据）
     */
    public List<org.springframework.ai.document.Document> loadPdfByPage(Resource resource) {
        PagePdfDocumentReader reader = new PagePdfDocumentReader(resource,
                PdfDocumentReaderConfig.builder()
                        .withPageExtractedTextFormatter(ExtractedTextFormatter.defaults())
                        .withPagesPerDocument(1)
                        .build());
        return reader.get();
    }
}
```

#### 5.2 字符切分（替换 `CharacterSplitter`）

```java
// Before: documentProcessorClient.loadAndSplitDocument(file, CHARACTER_SPLITTER, maxLen, overlap)

// After:
List<Document> rawDocs = loader.load(new FileSystemResource(file));
TokenTextSplitter splitter = new TokenTextSplitter(maxSegmentLength, overlapLength, 5, 10000, true);
List<Document> chunks = splitter.apply(rawDocs);
```

Spring AI `TokenTextSplitter` 参数：
- `chunkSize`：对应 `maxSegmentLength`
- `minChunkSizeChars`：最小块大小，建议 `overlapLength / 2`
- `minChunkLengthToEmbed`：过短内容忽略的阈值
- `keepSeparator`：保留分隔符

#### 5.3 章节切分（保留自研逻辑，适配 Spring AI Document）

```java
/**
 * 章节切分器 - 实现 Spring AI DocumentTransformer 接口
 */
public class ChapterDocumentTransformer implements DocumentTransformer {

    private final ChapterSplitter legacySplitter;  // 复用现有逻辑

    @Override
    public List<Document> apply(List<Document> documents) {
        // 从 Spring AI Document 获取内容，调用 legacySplitter 切分
        // 将切分结果封装回 Spring AI Document
        return documents.stream()
                .flatMap(doc -> splitToChapters(doc).stream())
                .toList();
    }
}
```

#### 6.1 RAG 检索改造（过渡方案）

将 `ChatController.enhancePromptWithKnowledge` 的底层检索替换为 Spring AI `VectorStore.similaritySearch()`，同时保留多库聚合逻辑：

```java
private String enhancePromptWithKnowledge(String originalPrompt, AppConfig appConfig) {
    List<Long> activeDatasetIds = parseActiveDatasetIds(appConfig.getDatasets());
    if (activeDatasetIds.isEmpty()) return originalPrompt;

    List<org.springframework.ai.document.Document> allResults = activeDatasetIds.stream()
            .flatMap(datasetId -> {
                try {
                    Dataset dataset = datasetService.getById(datasetId);
                    Provider provider = providerService.getProviderByModelId(dataset.getEmbeddingModelId());
                    Model model = modelService.getById(dataset.getEmbeddingModelId());

                    VectorStore vectorStore = vectorStoreRegistry.getOrCreate(
                            datasetId, dataset.getTenantId(), provider, model);

                    SearchRequest request = SearchRequest.query(originalPrompt)
                            .withTopK(5)
                            .withSimilarityThreshold(0.5)
                            .withFilterExpression("dataset_id == " + datasetId);

                    return vectorStore.similaritySearch(request).stream();
                } catch (Exception e) {
                    log.error("从知识库 {} 检索失败", datasetId, e);
                    return Stream.empty();
                }
            })
            .sorted(Comparator.comparingDouble(
                    doc -> -(double) doc.getMetadata().getOrDefault("distance", 0.0)))
            .limit(10)
            .toList();

    if (allResults.isEmpty()) return originalPrompt;

    StringBuilder context = new StringBuilder("请基于以下知识库内容回答用户问题：\n\n【知识库内容】\n");
    for (int i = 0; i < allResults.size(); i++) {
        context.append(String.format("%d. %s\n", i + 1, allResults.get(i).getContent()));
    }
    context.append("\n【用户问题】\n").append(originalPrompt)
           .append("\n\n请根据上述知识库内容回答用户问题。如果知识库内容与问题不相关，请直接回答用户问题。");

    return context.toString();
}
```

## 八、风险与注意事项

### 8.1 Qdrant 通信协议切换（REST → gRPC）

**风险**：现有自研代码通过 REST API（端口 6333）访问 Qdrant，Spring AI 默认使用 gRPC（端口 6334）。

**措施**：
1. 确认开发/生产环境 Qdrant 实例已开启 gRPC 端口（`--service.grpc_port 6334`）
2. 确认 Qdrant 版本 ≥ 1.x（Spring AI 使用的 Qdrant gRPC Client 需较新版本）
3. 迁移完成后，**不需要迁移现有向量数据**，集合命名规则不变

### 8.2 向量维度一致性

智谱 `embedding-3` 模型支持 2048 维，现有 Qdrant 集合按此维度创建。`ZhipuAiEmbeddingModel` 默认维度需通过 `ZhipuAiEmbeddingOptions` 显式配置：

```java
ZhipuAiEmbeddingOptions.builder()
    .model("embedding-3")
    .dimensions(2048)   // 必须与现有集合维度一致
    .build();
```

### 8.3 Spring AI Document ID 与现有 vectorId 的对应

Spring AI `Document` 的 `id` 字段对应 Qdrant Point 的 `id`。现有 `DatasetDocumentSegment.vectorId` 是 UUID 字符串，与 Spring AI `Document` 的 `String id` 兼容——迁移时**直接将 `vectorId` 作为 `new Document(vectorId, content, metadata)` 的 id**，无需数据迁移。

### 8.4 Spring AI 相似度分数差异

Spring AI `VectorStore.similaritySearch()` 返回的 `Document.getMetadata()` 中包含 `"distance"` 字段（Qdrant 余弦相似度 = 1 - distance），与现有 `QueryResult.score` 字段语义需对齐：

```java
// 从 Spring AI Document 获取相似度分数
double score = (double) doc.getMetadata().getOrDefault("distance", 0.0);
// 注意：Qdrant 余弦距离中，score 越高越相似（0~1）
```

### 8.5 多租户 Bean 生命周期

Spring AI 自动配置的 `ZhipuAiChatModel` / `ZhipuAiEmbeddingModel` Bean 是单例，而我们通过 Registry 手动创建多个实例。需注意：

- **不要将** Registry 中的实例注入 Spring 容器（不加 `@Bean`），避免 Bean 数量无限增长
- Registry 使用 `ConcurrentHashMap` + 过期策略管理，避免内存泄漏
- Provider API Key 变更后需主动调用 `registry.evict(provider)` 清理缓存

### 8.6 渐进式迁移避免大爆炸

各阶段改造**不需要**同时提交，可在分支上逐阶段进行并单独测试。建议顺序：

```
第一阶段（依赖引入）→ 第三阶段（Embedding）→ 第四阶段（VectorStore）→ 第五阶段（文档处理）→ 第二阶段（Chat）→ 第六阶段（清理）
```

先改造独立性强的 Embedding + VectorStore，后改造与业务耦合更深的 Chat + RAG。
