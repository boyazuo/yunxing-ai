# 17. 本地 Ollama 嵌入模型集成方案

## 1. 背景与目标

当前嵌入模型使用智谱 AI 的 `embedding-3`（云端 API，维度 2048），每次向量化文档或执行语义检索都需要消耗 API 调用额度。

目标：将嵌入模型切换为本地运行的 Ollama + `qwen3-embedding:4b`，实现：

- **零调用成本**：嵌入向量化完全在本地完成，无需 API Key
- **低延迟**：省去网络往返，本地推理响应更快
- **数据私密**：文档内容不出本地环境

本方案适用于**开发/测试环境**的嵌入模型配置，生产环境可按同样架构对接其他兼容 Ollama 协议的模型服务。

---

## 2. 前置确认：验证向量维度

**qwen3-embedding:4b 的默认输出维度为 2560**，切换前必须先本地验证，因为维度不匹配会导致 Qdrant 集合无法复用。

运行以下命令验证 Ollama 是否就绪并确认维度：

```bash
# 1. 确认 Ollama 服务正常
curl http://localhost:11434/api/embeddings \
  -d '{
    "model": "qwen3-embedding:4b",
    "prompt": "今天天气很好，适合出门散步。"
  }' | python3 -c "import sys,json; data=json.load(sys.stdin); print('维度:', len(data['embedding']))"
```

预期输出：`维度: 2560`

> **说明**：如果输出维度不是 2560，需在第 4 步更新 `dimensions` 配置值。Qwen3-Embedding 支持 Matryoshka 表示学习（MRL），可通过 Ollama 参数指定截断维度（256/512/1024），但默认使用完整维度。

## 3. 技术分析

### 3.1 现有嵌入模型架构

```
application-dev.yml
  yxboot.ai.embedding.provider = zhipuai
  yxboot.ai.embedding.model    = embedding-3
  yxboot.ai.embedding.dimensions = 2048
        ↓
AiProperties.EmbeddingConfig
        ↓
SpringAIConfig.embeddingModel()
  → ZhiPuAiEmbeddingModel（全局单例 Bean）
        ↓
VectorStoreRegistry.createVectorStore()
  → QdrantVectorStore(collectionName, embeddingModel, initializeSchema=true)
     集合维度由 EmbeddingModel.dimensions() 决定
        ↓
Qdrant 集合：tenant_{tid}_dataset_{did}，向量维度 2048
```

### 3.2 变更影响分析

| 影响点 | 说明 | 处理方式 |
|--------|------|----------|
| **Maven 依赖** | 需新增 Ollama starter | 添加 `spring-ai-starter-model-ollama` |
| **AiProperties** | 新增 `baseUrl` 字段 | 用于配置 Ollama 服务地址 |
| **SpringAIConfig** | `switch` 分支缺少 `ollama` | 添加 Ollama 构建逻辑 |
| **application-dev.yml** | embedding 配置需更新 | 切换 provider/model/dimensions |
| **Qdrant 集合** | 维度从 2048 → 2560（不兼容） | 删除旧集合，重新向量化 |
| **dataset.embedding_model** | 数据库中已记录的模型标识 | 重新处理文档后自动更新 |

### 3.3 不受影响的部分

- `VectorStoreRegistry`：无需改动，`initializeSchema=true` 会自动用新维度创建集合
- `AiVectorStoreService`、`DatasetDocumentProcessingApplicationService`：无需改动
- 向量检索逻辑（`ensureEmbeddingModelCompatible`）：模型标识变更后自动检测

## 4. 实现步骤

### Step 1：添加 Maven 依赖

编辑 `api/pom.xml`，在 Spring AI 依赖块中新增：

```xml
<!-- Spring AI -->
<dependency>
    <groupId>org.springframework.ai</groupId>
    <artifactId>spring-ai-starter-model-zhipuai</artifactId>
</dependency>
<!-- 新增：Ollama 本地模型支持 -->
<dependency>
    <groupId>org.springframework.ai</groupId>
    <artifactId>spring-ai-starter-model-ollama</artifactId>
</dependency>
<dependency>
    <groupId>org.springframework.ai</groupId>
    <artifactId>spring-ai-starter-vector-store-qdrant</artifactId>
</dependency>
```

> **说明**：保留 `zhipuai` 依赖，确保 Chat 模型不受影响。`spring-ai-starter-model-ollama` 由 `spring-ai-bom 1.1.6` 统一管理版本，无需手动指定。

---

### Step 2：修改 AiProperties.java

在 `EmbeddingConfig` 中新增 `baseUrl` 字段，支持配置 Ollama 服务地址。

**文件**：`api/src/main/java/com/yxboot/ai/config/AiProperties.java`

```java
@Data
public static class EmbeddingConfig {
    private String provider = "zhipuai";
    private String apiKey;
    private String model = "embedding-3";
    private Integer dimensions = 2048;
    /** Ollama 专用：服务地址，默认 http://localhost:11434 */
    private String baseUrl = "http://localhost:11434";

    /** 返回向量模型标识，格式 provider:model */
    public String toModelKey() {
        return provider + ":" + model;
    }
}
```

---

### Step 3：修改 SpringAIConfig.java

在 `embeddingModel()` 方法的 `switch` 中新增 `ollama` 分支。

**文件**：`api/src/main/java/com/yxboot/ai/config/SpringAIConfig.java`

新增 import：

```java
import org.springframework.ai.ollama.OllamaEmbeddingModel;
import org.springframework.ai.ollama.api.OllamaApi;
import org.springframework.ai.ollama.api.OllamaOptions;
```

修改 `embeddingModel()` 方法：

```java
@Bean
@ConditionalOnMissingBean
public EmbeddingModel embeddingModel(AiProperties props) {
    AiProperties.EmbeddingConfig cfg = props.getEmbedding();
    return switch (cfg.getProvider().toLowerCase()) {
        case "zhipuai", "zhipu" -> buildZhiPuEmbeddingModel(cfg);
        case "ollama" -> buildOllamaEmbeddingModel(cfg);
        default -> throw new IllegalArgumentException("不支持的 Embedding 提供商: " + cfg.getProvider());
    };
}
```

新增 `buildOllamaEmbeddingModel` 私有方法：

```java
private EmbeddingModel buildOllamaEmbeddingModel(AiProperties.EmbeddingConfig cfg) {
    String baseUrl = StringUtils.hasText(cfg.getBaseUrl())
            ? cfg.getBaseUrl()
            : "http://localhost:11434";
    OllamaApi ollamaApi = new OllamaApi(baseUrl);
    return OllamaEmbeddingModel.builder()
            .ollamaApi(ollamaApi)
            .defaultOptions(OllamaOptions.builder()
                    .model(cfg.getModel())
                    .build())
            .build();
}
```

> **说明**：Ollama 本地服务不需要 API Key，因此无需校验 `apiKey` 是否存在。`OllamaOptions` 只需指定 `model`，Ollama 服务端负责管理模型加载。

### Step 4：更新配置文件

### Step 5：清理旧 Qdrant 数据并重新向量化

**这是最关键的迁移步骤。** 由于向量维度从 2048 变更为 2560，旧的 Qdrant 集合无法继续使用，必须删除并重建。

#### 5.1 确认现有知识库数量

```sql
-- 查询当前所有知识库及其嵌入模型状态
SELECT id, tenant_id, name, embedding_model, status
FROM dataset
WHERE deleted = 0
ORDER BY tenant_id, id;
```

记录下所有知识库的 `tenant_id` 和 `id`，用于后续验证。

#### 5.2 停止后端服务

```bash
# 停止正在运行的后端服务，防止向量化操作中断
```

#### 5.3 清除数据库中的向量 ID 和模型标识

```sql
-- 清空 dataset.embedding_model（触发模型漂移检测，标记需重新向量化）
UPDATE dataset SET embedding_model = '' WHERE deleted = 0;

-- 清空所有文档分段的 vector_id（确保重处理时不使用旧 ID）
UPDATE dataset_document_segment SET vector_id = NULL
WHERE deleted = 0;

-- 将文档状态重置为"待处理"（触发重新向量化流程）
UPDATE dataset_document
SET status = 'waiting'
WHERE deleted = 0 AND status = 'completed';
```

> **状态值说明**：`waiting` 是 `DatasetDocument` 的初始状态，后端文档处理服务会自动拾取并处理。

#### 5.4 删除 Qdrant 中的所有旧集合

通过 Qdrant HTTP API 删除旧集合（也可在管理界面操作）：

```bash
# 列出所有集合
curl http://localhost:6333/collections | python3 -m json.tool

# 批量删除所有 tenant_*_dataset_* 集合（根据实际集合名替换）
# 示例：删除 tenant_1_dataset_1
curl -X DELETE http://localhost:6333/collections/tenant_1_dataset_1

# 如果有多个集合，逐一执行
# curl -X DELETE http://localhost:6333/collections/tenant_1_dataset_2
```

或使用 Qdrant Dashboard（默认地址 `http://localhost:6333/dashboard`）手动批量删除。

#### 5.5 重启后端服务并触发重新向量化

重启后端服务后，文档处理服务会检测到状态为 `waiting` 的文档，自动用新的 Ollama 模型重新向量化。可在日志中观察进度：

```
INFO  - 批量向量化完成, datasetId=1, tenantId=1, count=25
INFO  - recordEmbeddingModel: datasetId=1, model=ollama:qwen3-embedding:4b
```

#### 5.6 验证迁移结果

```sql
-- 确认所有文档已重新向量化完成
SELECT status, COUNT(*) AS cnt
FROM dataset_document
WHERE deleted = 0
GROUP BY status;
-- 期望：status='completed' 的数量 = 原 completed 数量

-- 确认知识库的嵌入模型标识已更新
SELECT id, name, embedding_model FROM dataset WHERE deleted = 0;
-- 期望：embedding_model = 'ollama:qwen3-embedding:4b'
```

## 6. 注意事项

### 6.1 Ollama 服务依赖

本地开发环境必须保持 Ollama 服务运行，否则嵌入向量化和语义检索都会失败。建议将 Ollama 设置为系统服务自启动：

```bash
# macOS：Ollama 安装后默认以 launchd 方式自启
# 手动启动
ollama serve

# 验证模型已拉取
ollama list
```

### 6.2 首次向量化性能

`qwen3-embedding:4b` 是 4B 参数模型，首次处理大量文档时，本地 CPU 推理较慢。如有 GPU 加速（Apple Silicon MPS 或 NVIDIA CUDA），Ollama 会自动利用。

### 6.3 维度不一致风险

> **警告**：如果跳过 Qdrant 集合清理步骤，直接切换模型启动服务，`initializeSchema=true` 会因新维度与已有集合不匹配而抛出异常，导致服务无法正常向量化文档。

### 6.4 生产环境考量

生产环境如需使用 Ollama，需：
1. 部署一台 GPU 服务器运行 Ollama
2. 将 `base-url` 配置为该服务器地址
3. 确保网络安全（建议内网访问，不对公网暴露）

### 6.5 回滚方案

如需回滚至智谱 AI 嵌入模型：

1. 将 `application-dev.yml` 的 embedding 配置恢复为：
   ```yaml
   embedding:
     provider: zhipuai
     api-key: <api-key>
     model: embedding-3
     dimensions: 2048
   ```
2. 重复执行第 5 步的数据清理流程（反向：维度 2560 → 2048）
